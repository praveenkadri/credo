# Wealthsimple Reference System — Exacting Guardrails

This file defines the target visual language.

The target is not generic premium.
The target is a close Wealthsimple-like look and feel:
- large open canvas
- left rail with almost no visual noise
- soft top utility band
- dominant chart / balance instrument
- low-contrast support modules
- editorial whitespace
- restrained semantic green for positive money movement
- minimal component chrome
- support content underneath the hero chart, not competing with it

## 1. Reference interpretation

Use the uploaded reference screenshots plus official Wealthsimple public imagery as the visual benchmark.

Reference traits:
- hero value is large and immediately visible
- positive delta sits directly below the hero value
- graph owns the largest share of the composition
- timeline controls are small and quiet
- lower sections feel like account rows or lightweight modules, not dashboard cards
- right-side support areas are useful but visually secondary
- the canvas reads as one calm field with lightly surfaced islands, not a card grid

## 2. Hard visual traits

- warm light-grey or cool off-white background only
- black / charcoal headline text
- low-contrast secondary text
- rounded surfaces, but not puffy
- almost invisible borders
- shadows used sparingly
- primary chart line must be the strongest colored element on the page
- green is allowed only where it signals positive movement

## 3. Exact layout behavior

When a hero graph exists:
- page hierarchy must be chart-first
- graph + balance block must dominate the main column
- right rail must never visually outweigh the graph
- lower modules must be flatter and lighter than the hero area
- shell chrome must be quieter than content

## 4. Exact control behavior

Controls should feel like WS controls:
- tiny range controls
- slim soft pills
- low contrast inactive state
- subtle active state
- almost no border visibility
- generous spacing around controls

Do not use:
- large segmented controls
- boxed filter bars
- strong dividers
- accent-heavy buttons in the hero area

## 5. Exact chart direction

A primary chart must include:
- large primary money value above the graph
- green positive delta under the value when up
- thin, elegant green line
- faded post-hover segment
- active point marker
- vertical hover guide
- subtle dotted / textured fill under active area
- timeline control row under or around the chart in the WS rhythm
- optional secondary metric at the top-right if the screen needs it

## 6. Anti-patterns

Never allow:
- equal visual weight across modules
- chart inside nested cards
- right rail as a stack of same-looking cards
- blue or grey-only positive chart line
- harsh tooltip balloons
- dense tables in hero areas
- enterprise dashboard spacing
- utility-first visual clutter

## 7. Final gate

If a screen does not immediately evoke:
- balance-first
- chart-first
- calm wealth / money operating surface

then it is not aligned enough.
