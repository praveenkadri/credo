# Overview Page Spacing Lock

This file overrides generic shell spacing for the overview page.

Use this when implementing the main overview/dashboard screen.

---

## 1. Topbar must not feel like a floating band

For the overview page:

* reduce topbar top padding
* reduce detached-card feeling
* keep utility strip quiet and shallow
* keep it visually connected to the content column

Do not use:

* oversized floating top shell
* excessive top whitespace
* strong rounded-card feel in the header band

Preferred direction:

* lower chrome
* tighter vertical rhythm
* calmer utility layer

---

## 2. Workspace / greeting block must align with WS rhythm

The workspace label and title row should feel:

* compact
* lightly offset from the shell
* not like a separate feature block

Rules:

* keep eyebrow small and quiet
* keep workspace name compact
* reduce vertical gap between topbar and overview heading
* do not create a large dead zone before the hero chart

---

## 3. Overview hero spacing

The vertical rhythm should be:

1. top utility strip
2. workspace label
3. overview heading
4. support copy
5. hero graph

These must feel closely related.

Do not let:

* workspace row float too far above title
* title sit too far above supporting text
* hero graph drift too far below intro

---

## 4. Hero graph should start higher

On the overview page:

* pull the hero graph upward slightly
* reduce excess gap between intro copy and graph
* keep graph dominant and immediate

The chart should begin sooner in the reading flow.

---

## 5. Right rail alignment

The right rail should start near the hero area, not feel dropped lower or too detached.

Rules:

* align right rail top rhythm with the upper content zone
* keep its first module visually related to the hero graph region
* avoid excessive top offset

---

## 6. Lower section spacing

Operational summary, directory, and activity sections should have:

* clear spacing from the hero graph
* but not oversized empty bands

Use:

* tighter than generic airy shell spacing
* calmer than dashboard stacking
* one consistent rhythm

---

## 7. Final rule

If the page feels like:

* floating shell pieces with too much air
  instead of:
* one coherent WS-like content flow

then spacing is wrong and must be tightened.
